debug.log — a system trying its best

Experimental Publishing Part B · dataset publication / mundane interface parody

debug.log — a system trying its best publishes real error messages from my creative coding and electronics practice as a dataset, paired with tender reinterpretations. The layout is generated in p5.js as a 90s-style console log—kept intentionally restrained so it reads like an archival system artifact.

this project also functioned as an early experiment for my thesis trajectory (soft computer / tender errors), reframing failure as a record of effort rather than a problem to erase.

Tools

  • p5.js (layout as design system)
  • print workflow (riso / inkjet / laser)
  • dataset: collected errors from my own practice

What to look for

  • timestamps + log levels
  • tension between rigid format + soft language
  • archival presentation (log as artifact)

How to Touch Electricity

Experimental Publishing Part A · tutorial as art · basic zine + small kit

How to Touch Electricity (Soft Circuits for Soft Beings) is a simple tactile tutorial zine that teaches basic circuit ideas through gestures like pressing, tracing, and folding. It pairs the publication with a small physical kit so the reader can activate each circuit by touch.

Materials

  • copper tape
  • coin cell + led
  • graphite / paper traces
  • optional conductive fabric

Focus

  • touch as input
  • poetic reframing of components
  • tutorial as artwork

Process

A quick overview of how these pieces were made—print choices, code choices, and what I cared about.

Workflow

  1. collect / write source material (logs, text, sketches)
  2. use code to generate structure (layout, timestamps, formatting)
  3. export at print resolution
  4. print, assemble, test, iterate
  5. scan / photograph and archive

Documentation assets to include

  • 1 hero photo per project
  • 1–3 detail shots (close-ups, binding, texture)
  • 1 scan (flat, clean) if possible
  • code screenshot or link
  • short reflection (3–6 lines)

Throughline

Across these projects, I’m treating publishing as a way to make invisible systems legible—sometimes through tenderness, sometimes through parody, often through restraint. The goal is clarity without losing feeling.
